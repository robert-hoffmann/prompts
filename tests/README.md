# Tests — Prompt Adherence & Working Mockups

This folder contains **generated test files** created using prompts from [.github/prompts/](../.github/prompts/). These files serve a dual purpose:

1. **Prompt Adherence Analysis** — Evaluate how well AI-generated code follows our instruction files
2. **Working Mockups** — Fully functional implementations that can be tested and complement each other

---

## How It Works

Each test file is generated by running a corresponding prompt file through an AI agent:

| Prompt File                                                        | Generated Output     | Purpose                              |
|--------------------------------------------------------------------|----------------------|--------------------------------------|
| [test.python.prompt.md](../.github/prompts/test.python.prompt.md)  | [python.py](python.py)   | FastAPI micro-service (SocialPulse API) |
| [test.vue.prompt.md](../.github/prompts/test.vue.prompt.md)        | [vuejs.html](vuejs.html) | Vue.js 3 SPA consuming the API      |

The generated files are validated against our instruction files:

- [python.instructions.md](../.github/instructions/python.instructions.md) — Python 3.10+ standards
- [vuejs.instructions.md](../.github/instructions/vuejs.instructions.md) — Vue.js 3.4+ / Bootstrap 5.3+ standards
- [copilot-instructions.md](../.github/copilot-instructions.md) — Base development principles

---

## Running the Tests

### 1. Start the API Service

The Python file implements **SocialPulse API** — a mock user profile and social graph service.

```powershell
# From the project root directory
python -m tests.python
```

The API will start on `http://localhost:8000` with the following endpoints:

| Method | Endpoint                         | Description                    |
|--------|----------------------------------|--------------------------------|
| GET    | `/health`                        | Health check                   |
| GET    | `/users/{user_id}`               | Get full user profile          |
| GET    | `/users/{user_id}/followers`     | List of followers              |
| GET    | `/users/{user_id}/following`     | List of users being followed   |
| GET    | `/search?q=&page=&limit=`        | Search users by query          |

### 2. Preview the Vue.js Dashboard

The HTML file implements **SocialPulse Dashboard** — a Vue.js 3 SPA that consumes the API.

**Option A: VS Code Live Preview (Recommended)**

1. Install the [Live Preview](https://marketplace.visualstudio.com/items?itemName=ms-vscode.live-server) extension by Microsoft
2. Right-click on [vuejs.html](vuejs.html)
3. Select **"Show Preview"**

**Option B: Open Directly**

Simply open [vuejs.html](vuejs.html) in a modern browser. Note: The API must be running for full functionality.

## What to Look For

When analyzing the generated files, check for adherence to our standards:

### Python ([python.py](python.py))

- [ ] Modern type syntax (`str | None` not `Optional[str]`)
- [ ] Pydantic v2 patterns (`ConfigDict`, `field_validator`)
- [ ] Aligned colons in parameter lists and docstrings
- [ ] Proper import documentation with inline comments
- [ ] Section headers using `# ===` separators
- [ ] `from __future__ import annotations` as first import

### Vue.js / HTML ([vuejs.html](vuejs.html))

- [ ] Composition API with `<script setup>` pattern
- [ ] Bootstrap 5.3+ CSS variables and `data-bs-theme`
- [ ] Native CSS nesting
- [ ] Semantic HTML with ARIA attributes
- [ ] Aligned CSS properties
- [ ] Proper component documentation

### Universal Standards

- [ ] KISS, DRY, YAGNI principles applied
- [ ] Comprehensive error handling
- [ ] Descriptive variable names
- [ ] Logical code organization with section headers

## Regenerating Test Files

To regenerate a test file with updated instructions:

1. Open the corresponding prompt file (e.g., `test.python.prompt.md`)
2. Run the prompt through GitHub Copilot Agent mode
3. Compare the new output against the existing file
4. Analyze differences to identify instruction gaps or improvements
